================================================================================
  PHASE 3: TRAJECTORY EVALUATION (AGENT LOOP EFFICIENCY)
================================================================================

  Status: IMPLEMENTED ✅

  [x] 3.1 Define Optimal Trajectories
      - 8 trajectory test cases with optimal tool chains defined
      - Covers: single-tool, two-step RAG, three-step search+read+query
      - File: tests/eval/trajectory_cases.json

  [x] 3.2 Build Trajectory Scorer (eval_trajectory.py)
      - Runs agent and captures FULL tool call chain (not just first call)
      - Scores efficiency using weighted formula:
          40% Step Efficiency (optimal_steps / actual_steps)
          40% Tool Overlap (correct tools used)
          -15% per error (error penalty)
          -10% per wasted call (waste penalty)
      - File: tests/eval/eval_trajectory.py

  [x] 3.3 Error Recovery Analysis
      - Tracks errors during tool execution
      - Classifies as "🔄 Recovered" or "💥 Failed" based on final score
      - Detects: rate limits, XML hallucinations, max-step loops


================================================================================
  PHASE 4: RAG EVALUATION (CHROMADB RETRIEVAL QUALITY)
================================================================================

  Status: IMPLEMENTED ✅

  [x] 4.1 Install RAGAS Framework
      - Installed via: uv add ragas
      - RAGAS = Retrieval Augmented Generation Assessment

  [x] 4.2 Create RAG Test Dataset
      - 3 landmark papers with 12 questions and verified ground-truth answers
      - Papers: "Attention Is All You Need", "BERT", "GPT-3"
      - File: tests/eval/rag_test_cases.json

  [x] 4.3 Measure RAG Metrics (Custom + RAGAS)
      - Custom Metrics (free, no LLM tokens):
          Context Precision: % of retrieved chunks relevant to ground truth
          Context Recall: % of ground truth keywords found in retrieved chunks
          Hit Rate: Does any chunk contain the core answer?
      - RAGAS Metrics (optional, needs OpenAI key):
          Faithfulness, Answer Relevancy (enabled with --use-ragas flag)
      - File: tests/eval/eval_rag.py

  [x] 4.4 Tune Chunking Parameters
      - Configurable via CLI flags: --chunk-size and --chunk-overlap
      - Run A/B tests:
          python -m tests.eval.eval_rag --chunk-size 500 --chunk-overlap 100
          python -m tests.eval.eval_rag --chunk-size 1500 --chunk-overlap 300
      - Compare reports in tests/eval/reports/ to find optimal settings

  🔴 P0 — TRACK 1: HUMAN-IN-THE-LOOP (HITL) CONTROL COMPLETE ✅
================================================================================

  Effort: 2–3 days
  Impact: Huge — most impressive agentic pattern for interviews

  Currently the Researcher autonomously hands off to the Writer with no
  user approval. This track adds an interrupt-based review gate so the
  user can approve, revise, or abort before the Writer begins.

  [x] 1.1 Add human_review node to LangGraph
      - Use LangGraph's interrupt_before=["human_review"]
      - Graph pauses after Researcher finishes
      - File: src/ai_researcher/agent/graph.py

  [x] 1.2 Extend AgentState for approval tracking
      - Add human_approval: str ("pending", "approved", "revise")
      - Add revision_instructions: str for user feedback
      - File: src/ai_researcher/agent/state.py

  [x] 1.3 Build approval UI in Streamlit
      - Show research summary with Approve / Revise / Abort buttons
      - On "Revise": loop back to Researcher with user instructions
      - On "Approve": continue to Writer
      - File: src/ai_researcher/ui/streamlit_app.py

  [x] 1.4 Update CLI mode for HITL
      - Terminal prompt: "Approve this research? [y/n/revise]: "
      - File: src/ai_researcher/cli.py
